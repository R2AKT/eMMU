# 📘 MEGA-580 OS — Техническая документация для GitHub

> **Статус проекта:** Активная разработка  
> **Архитектура:** КР580ВМ80А (i8080-совместимая)  
> **Платформа:** Собственная шина eMMU РЕВ_2.0 с сегментно-страничной памятью  
> **Дата:** 20 июля 2026 г.

---

## ⚠️ Важное примечание

> **Подсистема системных вызовов (SYSCALL_TABLE) находится в стадии активной переработки.**  
> В текущей сборке таблица `SYSCALL_TABLE` содержит только заглушку `syscall_bad_number`. Реализация полного набора POSIX-сисколлов (`sys_exit`, `sys_open`, `sys_read`, `sys_write`, `sys_gettimeofday` и др.) запланирована в ближайших коммитах. Вся остальная инфраструктура ядра (VFS, RTC, NVRAM, TTY, LPT, планировщик) полностью функциональна и готова к интеграции.

## 🎯 Обзор проекта

**MEGA-580 OS** — монолитная операционная система реального времени, написанная полностью на ассемблере КР580ВМ80А (~3500 строк). ОС поддерживает:

- 🔄 **Вытесняющую многозадачность** с динамическими приоритетами и aging
- 📁 **Виртуальную файловую систему (VFS)** с таблицей коммутации драйверов
- 🕐 **Аппаратные часы реального времени (RTC)** на базе КР512ВИ1
- 💾 **Энергонезависимую память (NVRAM)** 50 байт
- 🖥️ **До 4-х независимых TTY-консолей** (автодетект КР580ВВ51А / NS16550)
- 🖨️ **До 2-х параллельных портов LPT** (автодетект КР580ВВ55А / Intel 82C11)
- 🛡️ **Защиту памяти** через eMMU с режимом `KERNEL_OVERRIDE`

## 🏗️ Архитектура ядра

### Холодный старт (Cold Boot)

Процесс загрузки реализован в 4 этапа:

1. **Теневое ПЗУ** → выполнение с адреса `0x0000`
2. **Перенос ядра** в верхнее ОЗУ (`EMMU_PAGE_2`, адрес `0x8000`)
3. **Отключение ПЗУ** через порт `0x75` и зеркальный перенос ядра в нижнее ОЗУ (`0x0000`)
4. **Чистый старт** из нижнего ОЗУ с вызовом `kernel_clean_start`

```asm
LXI  H, upper_ram_stage2 + EMMU_PAGE_2
PCHL                        ; Программный прыжок в верхнее ОЗУ
```

### Последовательность инициализации (`kernel_clean_start`)

| Этап | Функция | Назначение |
|------|---------|------------|
| 5.1 | `k_pcb_subsys_init` | Разметка `GLOBAL_RESOURCE_TABLE` и слотов PCB |
| 5.2 | `k_tty_subsys_init` | Очистка буферов `TTY_TABLE` |
| 5.3 | `k_lp_subsys_init` | Очистка буферов `LP_TABLE` |
| 5.4 | `k_vfs_subsys_init` | Зануление `SYS_FILE_TABLE` и `VFS_SWITCH_TABLE` |
| 5.5 | `hw_sys_init` | Инициализация системного таймера КР580ВИ53 (квант 20 мс) |
| 5.6 | `k_rtc_boot_sync_unix_time` | Синхронизация `SYS_UNIX_TIME` с аппаратным RTC |
| 5.7 | `k_tty_carousel_auto_build` | Автодетект TTY-контроллеров с SMC-патчингом ISR |
| 5.8 | `vfs_mount_tty_nodes` | Монтирование `/dev/tty0..tty3` в RAM-диск |
| 5.9 | `k_lp_carousel_auto_build` | Автодетект LPT-контроллеров |
| 5.10 | `vfs_mount_lp_nodes` | Монтирование `/dev/lp0..lp1` в RAM-диск |
| 5.11 | `hw_int_init` | Инициализация контроллера прерываний КР580ВН59 |
| 6 | `EI` + `idle_loop` | Запуск многозадачной карусели |

## 🕐 Подсистема RTC/NVRAM

### Полный двусторонний стек

Реализована уникальная для 8-битных систем конвертация между сырыми регистрами КР512ВИ1 и 32-битным POSIX `UNIX_TIME`:

```
k_rtc_raw_sec..year  ←→  k_rtc_convert_raw_to_unix  ←→  SYS_UNIX_TIME (32 bit)
                            k_rtc_convert_unix_to_raw
```

### Ключевые особенности

- **Защита от бита UIP:** Чтение регистров только в безопасное окно 244 мкс
- **Тест через Регистр B:** Исключение ложных срабатываний от динамического бита UIP
- **Оптимизированная арифметика:** Умножение дней на 86400 через разложение `65536 + 20864`
- **Таблица `k_rtc_days_before_month`** в формате `.dw` (16 бит) для поддержки октября/ноября/декабря
- **Корректная обработка високосных лет:** `(год % 4 == 0) && (месяц > 2)`

### VFS-мосты

| Устройство | Major | Minor | Функции |
|------------|-------|-------|---------|
| `/dev/rtc` | 8 | 1 | `rtc_read_bridge`, `rtc_write_bridge` |
| `/dev/nvram` | 9 | 0 | `nvram_read_bridge`, `nvram_write_bridge` |

Все мосты используют атомарное управление `KERNEL_OVERRIDE` через порт `0x76` для безопасного доступа к памяти Ring 3.

## 📁 Виртуальная файловая система (VFS)

### Диспетчеризация драйверов (`k_vfs_dispatch`)

Использует **оптимизацию хвостового вызова** через трюк `XTHL`:

```asm
LHLD VFS_DRV_ROW_PTR        ; HL = Адрес структуры файла (ABI драйвера)
PUSH H                      ; Сохраняем HL
LHLD K_TMP_PC               ; HL = Адрес функции драйвера
XTHL                        ; МАГИЯ: адрес функции на вершине стека, HL восстановлен
RET                         ; Прыжок в драйвер с чистыми РОН
```

### Разрешение путей (`k_vfs_resolve_dev`)

- Побайтовое сравнение ASCIIZ-строк Ring 3 с эталоном Ring 0
- **Файрвол длины:** Лимит 64 символа через `K_TMP_STR_LEN`
- **Математика стека:** Извлечение `DE_user` через `LXI H, 10` / `DAD SP`
- **Защита пустых слотов:** Проверка `D|E == 0` до чтения Ring 3

### Структура `SYS_FILE_TABLE` (10 байт)

| Смещение | Поле | Размер |
|----------|------|--------|
| +0 | `F_TYPE` (Major) | 1 байт |
| +1 | `F_MINOR` | 1 байт |
| +2 | `F_REF` (счётчик ссылок) | 1 байт |
| +3 | `F_INODE` | 2 байта |
| +5 | `F_OFFSET` (позиция) | 4 байта |
| +9 | `F_DIRTY` | 1 байт |

## 🔄 Планировщик задач

### Динамические приоритеты с aging

- **Штрафной счётчик:** `O_CPU_TICKS` увеличивается при каждом кванте выполнения
- **Формула приоритета:** `DYNAMIC = STATIC + NICE + CPU_TICKS` (с насыщением до 255)
- **Aging:** `O_CPU_TICKS--` для всех процессов, кроме `ST_ZOMBIE`
- **Выбор кандидата:** Линейный скан 32 слотов `PCB_TABLE` с поиском минимума

### Обработка будильников (POSIX `alarm`)

Каждый квант (20 мс) ISR сканирует `O_ALARM_TICKS` всех процессов. При достижении нуля — доставка сигнала `SIGALRM` в старший байт маски `O_SIG_PENDING`.

## 🖥️ Подсистема TTY

### Автодетект каруселью

Для каждого из 4-х портов (`0xA0`, `0xB0`, `0xC0`, `0xD0`) выполняется:

1. **Приоритетный тест КР580ВВ51А** + КР580ВИ53 (запись/чтение `0xAA55` в канал 0)
2. **Резервный тест NS16550** (Scratch Test: запись/чтение `0x5A` в регистр `REG_SCR`)
3. **SMC-патчинг ISR:** Прошивка адреса выделенного TTY-слота в операнд `LXI D`

```asm
MOV  A, E
STA  _smc_isr_51_base + 1   ; Младший байт операнда LXI D
MOV  A, D
STA  _smc_isr_51_base + 2   ; Старший байт операнда LXI D
```

### Кольцевые буферы

- Размер: 64 байта (`TTY_BUF_SIZE`)
- Индексы: `RX_HEAD`, `RX_TAIL`, `TX_HEAD`, `TX_TAIL`
- Счётчик: `RX_COUNT`, `TX_COUNT`
- **Атомарность:** Все операции чтения/записи под `DI`/`EI`

---

## 🖨️ Подсистема LPT

### Автодетект

- **Приоритетный тест Intel 82C11** (запись/чтение маски `IRQ_EN | BIDIR`)
- **Резервный тест КР580ВВ55А** (режим 0, шахматные маски `0x55` / `0xAA`)

### FIFO-буфер отложенной записи

- Размер: 48 байт
- Смещение в структуре: `+21`
- **Асинхронная выгрузка:** ISR формирует импульс STROBE через BSR-команды РРУ

### Handshake Centronics

```asm
MVI  A, 000H                 ; STROBE = 0 (активный уровень)
OUT  000H                    ; Патчится на BASE+3
NOP / NOP / NOP / NOP        ; Микровыдержка по временной диаграмме
MVI  A, 001H                 ; STROBE = 1 (пассивный уровень)
OUT  000H
```

## 🛡️ Безопасность и защита памяти

### eMMU РЕВ_2.0

- **4 сегмента** по 16 КБ (`EMMU_PAGE_SIZE = 4000H`)
- **Ring 0 / Ring 3** разделение через порт `0x76` (`EMMU_OVERRIDE`)
- **Порты управления:** `0x70..0x73` (регистрация сегментов), `0x75` (отключение ПЗУ)

### Обработчик нарушения привилегий (`hw_privilege_violation_entry`)

При попытке Ring 3 выполнить `IN`/`OUT`/`EI`/`DI`:
1. Перевод процесса в статус `ST_ZOMBIE`
2. Освобождение физических страниц через `k_free_physical_page`
3. Закрытие всех открытых файлов
4. Переключение контекста на живой процесс через `switch_mmu_context`
